## 指摘事項（重大度順）

### 1. High: Heartbeat の `taskLastRun` 更新漏れで再実行ループが起きうる

- 事象:
  - 実行対象 `tasks` を計算していても、`taskLastRun` は LLM 返却 `results` の `taskId` のみ更新される。
  - LLM が一部 taskId を返し損ねると、そのタスクだけ毎 tick 再実行され続ける可能性がある。
- 影響:
  - API コスト増加
  - 通知スパム/重複処理
- 参照:
  - `src/core/heartbeatCommon.ts:114`
  - `src/core/heartbeatCommon.ts:142`
  - `src/core/heartbeat.ts:274`

### 2. High: `stopStreaming` が実質的なキャンセルになっていない

- 事象:
  - `abortRef.current = true` でループを抜けるだけで、`run()` 自体の中断は行っていない。
  - その後 `result.completed` を待機するため、停止後も完了待ちが継続する。
- 影響:
  - 「停止」操作の体感不一致
  - 長時間 `isStreaming` が true のまま残る可能性
- 参照:
  - `src/hooks/useAgentChat.ts:99`
  - `src/hooks/useAgentChat.ts:165`
  - `src/hooks/useAgentChat.ts:206`

### 3. Medium: 送信失敗時の会話永続化と `messageCount` 更新が不整合

- 事象:
  - 失敗時は UI 上で assistant エラーメッセージを表示するが、DB に保存しない。
  - 一方で会話 `touch` は通常送信想定の `+2` を前提として更新される。
- 影響:
  - リロード後に表示内容が消える
  - 表示メッセージ数と保存済み件数の乖離
- 参照:
  - `src/hooks/useAgentChat.ts:188`
  - `src/App.tsx:149`

### 4. Medium: Heartbeat 実行レイヤで利用ツールが不一致

- 事象:
  - フォアグラウンド（Layer1: `createHeartbeatAgent`）と Worker/SW（`executeWorkerTool`）で利用可能ツール群が一致していない。
  - 一部タスク定義は Worker 側ツール前提になっている。
- 影響:
  - タブ表示/非表示でタスク成功率や品質が変わる
  - 実行経路依存の不安定性
- 参照:
  - `src/core/agent.ts:125`
  - `src/core/config.ts:42`
  - `src/core/heartbeatTools.ts:802`

### 5. Medium: Heartbeat 応答パースが壊れやすい

- 事象:
  - 正規表現で JSON 部分抽出後に `JSON.parse` を前提。
  - 形式逸脱時の堅牢なフォールバックが弱い。
- 影響:
  - モデル出力の微小な揺れでチェック全体が失敗
- 参照:
  - `src/core/heartbeatOpenAI.ts:191`
  - `src/core/heartbeatOpenAI.ts:194`
  - `src/core/heartbeatOpenAI.ts:149`
